## 2. Database Modifications
### New Table
* **`operator_alerts`**: Stores active transit operator alerts regarding delays, closures, or maintenance.
  * Columns: `alert_id` (PK), `line`, `station_id`, `severity` (low, medium, high), `message`, `created_at`, `is_active`.

### Table Columns Updated
* **`national_rail_bookings`**: Modified to support logging and storing `departure_time` (TIME) per user booking rather than assuming a single fixed daily train.

## 3. Python Query Functions Added / Modified
* **`query_active_alerts()`**: Fetches all active operator alerts sorted by severity (high first).
* **`query_station_upcoming_departures(station_id)`**: Dynamically calculates and returns scheduled departures passing through a station based on frequency, first train time, and travel offset.
* **`query_transit_system_analytics()`**: Computes system-wide operational analytics including booking counts, revenue split by payment method, busiest stations, and user rating summaries.
* **`execute_booking(...)`**: Updated signature to accept `departure_time: Optional[str] = None`. It writes the custom departure time to the booking record if provided; otherwise, it defaults to the schedule's first train departure offset time.

## 4. AI Agent & Tool Enhancements
* Added **`check_operator_alerts`**, **`get_upcoming_departures`**, and **`get_system_analytics`** to tools.
* Extended the **`make_booking`** tool parameters to accept `departure_time`.
* Implemented parameter validation and cleanup logic in `skeleton/agent.py` to identify and discard malformed schema structures generated by Ollama (`llama3.2:1b`), successfully routing those queries to the deterministic fallback parser.

## 5. Gradio Frontend UI Enhancements
* Added a custom CSS file styling system enabling **glassmorphism**, dynamic hover effects, and custom alert badges.
* Integrated a high-end **gradient header banner** at the top of the interface.
* Restructured the left content panel into **four tabs**:
  1. **💬 AI Chat Assistant**: Original smart conversational helper.
  2. **⚠️ Service Alerts**: Displays live PostgreSQL alerts (high, medium, low) in styled warning cards.
  3. **🕒 Station Departures**: Computes daily scheduled departures dynamically for any selected station.
  4. **📊 System Analytics**: Renders cards for System Revenue, Bookings, Trips, and Ratings alongside top-station tables and payment details.
